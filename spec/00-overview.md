# 00 — Overview

## ゲーム概要

縦画面のブラウザゲーム「**Pull Strike**」。
モンスターストライクのコア体験 — *ひっぱって弾く / 壁で反射 / 敵に当たる / ターン制で次へ* — だけを抽出した最小実装。

- スマホ（タッチ）とマウスの両方で操作できる。
- ステージは 9:16 縦画面。内部解像度は **540 × 960 px** を基準とする。
- **AI / 自動テストから操作・観測**できることを最重要要件とする。

## スコープ（やる）

- ひっぱり方向の逆へ味方キャラを発射
- 壁での反射
- 円同士の衝突判定（プレイヤー × 敵）
- 反射タイプ / 貫通タイプの切替
- 速度に比例したダメージ
- 敵 HP が 0 で撃破、全滅でステージクリア
- ターン制（味方が止まったら次キャラへ）

## アウトオブスコープ（今はやらない）

- 敵ターンの能動的な攻撃（Phase 4 で導入予定）
- 友情コンボ（Phase 3）
- スキル・必殺技（Phase 5）
- マルチプレイ
- ガチャ・育成
- 公式キャラ・公式画像・公式名称（**永久に使わない**）

## 技術スタック

| レイヤー | 採用 | 理由 |
|---|---|---|
| ビルド | Vite | ESM・HMR・TS そのまま |
| 言語 | TypeScript (strict) | AI/人間どちらにも読みやすい |
| 描画 | Canvas 2D | 物理を自前で書く前提なら十分 |
| 物理 | 自前 | 反射/貫通/めり込み補正を完全に握りたい |
| UI | DOM (CSS) | HUD は state を読むだけで十分 |

将来の選択肢（必要になったら検討）:
- 大量スプライト / エフェクト → **PixiJS**
- ゲームエンジン全体の作法を学びたい → **Phaser**
- 剛体物理を試す → **Matter.js**

## 設計の柱

1. **状態を 1 か所に**: `GameState` だけが事実。描画も AI もここを読む。
2. **ロジック / 入力 / 描画を分離**: `Game` クラスは DOM も RAF も知らない。
3. **副作用は端に寄せる**: `physics.ts` は `bodies / events` を変更するだけの純ロジック寄り。
4. **決定論性**: 固定タイムステップ + シード可能な PRNG。同じ入力 → 同じ結果。
5. **観測可能性**: 全イベントを `GameEvent` 配列で蓄積。AI も人間も同じデータを読む。
