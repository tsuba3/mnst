# Pull Strike — 仕様書

このディレクトリは **「spec ファースト」** で運用します。

- コードの仕様変更が必要になったら、**先に該当する spec を編集** → その後コードを直す。
- AI（Claude / 他のエージェント）が実装する際も、**spec を真として参照**する。
- spec は人間とエージェントの両方が読む前提で、**自然言語 + 簡潔な擬似コード + JSON 例**で書く。

## 仕様書の変更ルール

1. spec を編集する PR では、**変更点と理由（Why）を冒頭に書く**。
2. spec の段落と実装コードの対応が崩れたら、コードを直すか、spec を直す。**両方ズレたまま放置しない**。
3. 仕様変更が破壊的な場合、`### 移行ノート` を追記する。
4. 数値（速度上限・減衰率など）を変えたら、**該当 spec の数値も同じコミットで更新**する。
